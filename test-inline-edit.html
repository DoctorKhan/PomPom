Todo Inline Edit Test

Test Instructions:

  1. Click "Open PomPom App" to open the main app in offline mode
  2. Go to the Planner view (calendar icon in sidebar)
  3. Add a todo item using the input field
  4. Click on the todo item text to enter inline edit mode
  5. Verify that input fields appear with correct placeholders
  6. Try editing the text and pressing Enter to save
  7. Try editing the due date field and pressing Enter to save
  8. Verify that changes are saved and displayed correctly
Open PomPom App (Offline Mode) Open PomPom App (Online Mode)

Expected Behavior:

  • Click to Edit: Clicking on any todo item should show inline edit inputs
  • Input Placeholders: Should see "Task description", "Owner (name)", "Due (e.g., 'tomorrow 5pm'...)", and "Tags (e.g., #bug urgent)"
  • Enter to Save: Pressing Enter in any input field should save the changes
  • Escape to Cancel: Pressing Escape should cancel editing and revert changes
  • Save Button: Clicking the Save button should also save changes
  • Text Selection: First click should select all text, second click should place caret
  • Natural Language: Due dates like "tomorrow 5pm", "in 2 days", "Friday 3pm" should work
  • Tags: Hashtags like #work #urgent should be extracted and displayed as tag pills

If the inline editing doesn't work as expected, check the browser console for errors.
